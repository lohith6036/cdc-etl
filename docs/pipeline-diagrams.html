STEP 01 — CAPTURE

PostgreSQL WAL Tailing

Debezium connects to Postgres via logical replication, reading the Write-Ahead Log to detect every INSERT, UPDATE, and DELETE without touching application code.

STEP 02 — STREAM

Kafka Event Queue

Change events are published to Kafka topics — one per table. Kafka guarantees durability, ordering, and fan-out to multiple consumers with configurable retention.

STEP 03 — PROCESS

Spark Structured Streaming

Spark consumes Kafka topics in micro-batches, applies schema transforms, deduplication, and upsert logic before writing to the downstream target.

STEP 04 — REPLICATE

Real-Time Sync to Target

Final data lands in the target store — a replicated Postgres DB, AWS Redshift, or S3-backed data lake — keeping it continuously in sync with the source.

STEP 01 — ORCHESTRATE

Airflow DAG Scheduling

Airflow defines the pipeline as a Directed Acyclic Graph (DAG). It schedules runs on a cron, monitors task health, retries failures, and triggers downstream tasks only when upstream succeeds.

STEP 02 — EXTRACT

Python Extraction Scripts

Python scripts query PostgreSQL in bulk — using incremental watermarks or full snapshots — extracting raw records and staging them for transformation.

STEP 03 — TRANSFORM

Spark Batch Processing

Spark runs distributed batch jobs: cleaning nulls, joining datasets, applying business rules, aggregating metrics, and reshaping data into target schemas at scale.

STEP 04 — LOAD

AWS Cloud Delivery

Processed data lands in AWS — S3 for raw/parquet storage, Redshift for analytics, or RDS for operational use. Docker + Kubernetes ensures reproducible, scalable deployment.
